# План: наглядный HTML-отчет после анализа чата

Дата: 2026-05-21

## Цель

Вернуть проект к главной продуктовой идее: бот/CLI анализирует Telegram-чат
про WB/Ozon и выдает человеку наглядную страницу, где видно, что найдено,
откуда взялись выводы, какие гипотезы построены и что нужно проверить дальше.

## Контекст

- Основной продукт не является одной гипотезой вроде `Stock Location Mismatch`.
- Основной продукт: локальный анализатор чатов с понятным визуальным отчетом.
- CLI и SQLite остаются движком обработки.
- HTML-страница становится главным результатом пилотного прогона.
- Сырой Telegram-экспорт, имена, handles, user IDs, приватные цитаты и URL не
  попадают в публичный отчет по умолчанию.
- Старый вывод `summary` полезен, но человеку трудно держать его в голове.

## Как должен выглядеть результат

Команда:

```powershell
python -m app.cli --db data/db/pilot-001.sqlite report --output data/reports/pilot-001.html
```

Страница должна показывать:

- сколько чатов, сообщений, меток и непроклассифицированных сообщений найдено;
- распределение категорий;
- частоты по ключевым типам сообщений;
- найденные кластеры боли/вопросов;
- найденные решения, рекламу и упоминания инструментов;
- карточки гипотез с оценкой, MVP и evidence IDs;
- кандидатов на ручную проверку;
- пробелы качества анализа;
- приватные источники только через безопасные aliases вроде `chat1:42`.
