# Retrospective: Phase 1 Thin Slice

Date/time: 2026-05-19 14:20

## Task

Собрать минимальный безопасный сценарий `import -> classify -> stats` на
синтетической Telegram-фикстуре без реальных людей, handles, user IDs и приватных
чатов.

## What Changed

- Добавлена синтетическая фикстура `tests/fixtures/telegram_result.json`.
- Добавлены тесты на покрытие формы экспорта, идемпотентный импорт, отсутствие
  дублей по `(chat_id, msg_id)`, обновление существующей записи и сохранение
  rule-based меток.
- Добавлена команда `python -m app.cli classify`.
- `stats` теперь показывает количество неклассифицированных сообщений и
  распределение сохраненных меток по категориям.
- Pytest настроен не заходить во временные `tests/_tmp*` каталоги и не писать
  cache-файлы в ограниченном окружении.

## Result

Фаза 1 выполнена: на чистой тестовой SQLite базе импортируется 8 синтетических
сообщений, сохраняется 8 меток, `stats` показывает `unclassified = 0` и
распределение по `pain`, `question`, `solution_ad`, `tool_mention`, `offtopic`.

## What We Learned

- Текущей таблицы `message_labels` достаточно для тонкого среза; поля
  `classifier_name`, `classifier_version`, `run_id` лучше добавить отдельным
  следующим шагом, вместе с версионированием качества.
- В песочнице Windows `pytest tmp_path` и `TemporaryDirectory` могут уходить в
  недоступные места, поэтому тесты используют явный временный SQLite-файл внутри
  `tests/`.
- От первой попытки с временными каталогами в окружении остались недоступные
  `tests/_tmp_phase1_*`; они игнорируются Git/pytest, но при возможности их стоит
  удалить вручную на машине.
